Installation

Installation has been tested using JavaTM Development Kit (JDKTM) 1.1.8 and JavaTM 2 Platform, Standard Edition, v 1.2.2 on Win32 and SolarisTM platforms. On the Mac, it was tested with MacOS 8.1 under MRJ 2.1.

Installation Notes

To install the JavaHelp 1.1 software, execute one of the self-extracting packages. These self-extracting packages are produced using InstallAnywhereTM technology. InstallAnywhere software provides a number of advantages:
  • An easy-to-use wizard-based installation/uninstall procedure
  • Automatically configured environment
  • The JavaHelp demo programs are launched using InstallAnywhere "LaunchAnywhere" technology. This enables the demos to be executed from the Win32 Start menu and executed from the execution path.
You can choose between two self-extracting installation packages for both the Solaris and Win32 platforms:
  • Packages with the "-rt" extension install the JavaHelp system plus a copy of the JavaTM 2 Runtime Environment (JRE) that is used by the demo programs and tools.
  • Packages with the "-no_rt" extension require that a JRE or JDK be already accessible on your system.

NOTES:
  • InstallAnywhere is a Java language application and requires JRE/JDK 1.1.6 and higher, or JRE/JavaTM 2 SDK to run. If a JRE/JDK is not already installed on your system, you MUST use the package that includes the JRE (-rt).

  • The MacOS version is not available with a packaged JRE ("-rt").

  • If you are unable to execute the self-extracting packages on your system, a Zip package is available for downloading at the JavaHelp website: 
       http://java.sun.com/products/javahelp

Installation Packages

The following installation files are included on the JavaHelp 1.1 CD:

Win32 Systems
javahelp1_0rc-win-rt.exe This self-extracting package installs the JavaHelp system plus a copy of the JavaTM 2 Runtime Environment that is used by the demo programs and tools. The JRE is installed in the folder named jre located in the JavaHelp system installation folder.
javahelp1_0rc-win-no_rt.exe This self-extracting package requires that a JavaTM 2 Runtime Environment be already accessible on your system. The demo programs and tools use that JRE/JDK. The JavaHelp software requires JRE/JDK 1.1.6 and higher or JRE/JDK 1.2 or higher. The installation software searches your execution path and asks you to choose a JRE/JDK.

Solaris/Unix Systems
javahelp1_0-solsparc-rt.bin This self-extracting package installs the JavaHelp system plus a copy of the Solaris JavaTM 2 Runtime Environment that is used by the demo programs and tools. The JRE is installed in the folder named jre located in the JavaHelp system installation folder.
javahelp1_0-unix-no_rt.bin This self-extracting package requires that a JavaTM 2 Runtime Environment be already accessible on your system. The demo programs and tools use that JRE/JDK. The JavaHelp software requires JRE/JDK 1.1.6 and higher or JRE/JDK 1.2 or higher. The installation software searches your execution path and asks you to choose a JRE/JDK.

This package should work for most Unix systems, but has only been tested on Solaris for SPARC and Linux (using the JRE/JDK from www.blackdown.org).

MacOS
javahelp1_0-macos-no_rt.bin This self-extracting package requires that a JavaTM 2 Runtime Environment be already accessible on your system. The demo programs and tools use that JRE/JDK. The JavaHelp software requires JRE/JDK 1.1.6 and higher or JRE/JDK 1.2 or higher. The installation software searches your execution path and asks you to choose a JRE/JDK.

Caveat For The Win32 Platform

File Separators ("/" vs. "\")

All files within the JavaHelp system are specified as URLs. The separator between elements (files) in a hierarchy should be "/". In some cases "\" works on the Win32 platforms, however when the files that contain these references are added to JAR files or moved to different platforms, these references no longer work.

Documentation

The contents of the release are described in the "Release Information" chapter of the JavaHelp System User's Guide. You can view the JavaHelp System User's Guide on Win32 by executing UserGuide from the Program menu (or whatever shortcut you specify during installation) or on Solaris by executing the UserGuide command in the demos/bin folder.

The JavaHelp System User's Guide is also included in PDF format and can be read and printed using the Adobe Acrobat Reader (doc\jhug\jhug.pdf).
